# ExpenseIQ — Project Context

ExpenseIQ is a full-stack personal finance tracker. Users log expenses against categories and budgets, then explore spending trends through interactive charts.

## Tech Stack

| Concern | Tool | Notes |
|---|---|---|
| Framework | Next.js 15 (App Router) | `src/` layout, strict TypeScript |
| Language | TypeScript 5 | strict mode, no `any` |
| Styling | Tailwind CSS v4 + shadcn/ui | Config in `postcss.config.mjs` |
| ORM | Prisma 7 (PostgreSQL) | Generator: `prisma-client` |
| Auth | NextAuth.js v5 (beta) | `auth()`, not `getServerSession` |
| Charts | Recharts | Wrapped in `src/components/charts/` |
| Forms | React Hook Form + Zod | `zodResolver`, infer types from schemas |
| Server state | TanStack Query v5 | Keys in `src/lib/queries/keys.ts` |
| Unit tests | Vitest | Colocated `*.test.ts(x)` |
| E2E tests | Playwright | `tests/e2e/` |
| Linting | ESLint + Prettier | `npm run lint`, `npm run format` |

## Architecture Decisions

### Server vs. Client Components
Default to **Server Components**. Add `"use client"` only for interactive elements (state, effects, event handlers). Data fetching and mutations belong in Server Components and server actions — never fire `fetch()` inside a Client Component.

### Mutations
Use **Next.js server actions** in `src/lib/actions/`. After a mutation, call `revalidatePath()` or `revalidateTag()`. Route Handlers (`app/api/`) are only for third-party webhooks and NextAuth — not for internal mutations.

### Forms
React Hook Form + `zodResolver`. Define the Zod schema in `src/lib/validations/<noun>Schema.ts`. Infer the TypeScript type with `z.infer<typeof schema>` — **do not** declare a separate interface.

### Auth — NextAuth v5
- Server component session: `const session = await auth();`
- Route protection: `middleware.ts` using the NextAuth `auth` export.
- **Never** use `getServerSession` (v4 API).
- OAuth and credentials providers both configured in `src/lib/auth.ts`.

### Database — Prisma 7
- Singleton client in `src/lib/db.ts` (prevents connection pool exhaustion on hot reload).
- Generator: `prisma-client` (Prisma 7 — **not** `prisma-client-js`). Output: `src/generated/prisma`.
- Migration naming: `verb-noun` (e.g., `add-expense-tags`, `rename-amount-column`).
- Never modify `prisma/migrations/` by hand.

### Styling — Tailwind v4 + shadcn/ui
- No `tailwind.config.js` — all config via CSS `@theme` directives in `src/app/globals.css`.
- Add new shadcn components: `npx shadcn@latest add <component>`.
- **Never** hand-edit `src/components/ui/` — always regenerate via the shadcn CLI.

## Code Conventions

- **Named exports only** — no default exports from component or action files.
- **File names**: `kebab-case.tsx` for components, `camelCase.ts` for utilities and actions.
- **No barrel `index.ts`** re-exports — import the concrete file directly.
- **Server actions**: named `verbNoun` (e.g., `createExpense`, `updateBudget`, `deleteCategory`).
- **Zod schemas**: named `<noun>Schema`; inferred TS type named `<Noun>` (e.g., `expenseSchema` → `type Expense`).
- **TanStack Query keys**: factory functions in `src/lib/queries/keys.ts`.
- **`data-testid`** attributes on all interactive elements (required by Playwright tests).

## Testing Strategy

| Layer | Tool | Location |
|---|---|---|
| Unit / component | Vitest + React Testing Library | Colocated `*.test.ts(x)` |
| Integration (DB) | Vitest + real test DB | `src/lib/actions/*.test.ts` |
| E2E | Playwright | `tests/e2e/` |

- **Do not mock Prisma** in integration tests — use a real test DB (`DATABASE_URL_TEST` env var). Wrap in transactions that roll back.
- Playwright tests reset state via a test-only seed before each suite.
- Coverage target: ≥ 80% for `src/lib/`.

## Environment Variables

See `.env.example`. Runtime requirements:

| Variable | Purpose |
|---|---|
| `DATABASE_URL` | Primary PostgreSQL connection string |
| `DATABASE_URL_TEST` | Isolated test DB (same schema, separate data) |
| `NEXTAUTH_URL` | Full app URL with protocol (e.g., `http://localhost:3000`) |
| `NEXTAUTH_SECRET` | 32-byte secret — `openssl rand -base64 32` |

## Known Constraints / Gotchas

- NextAuth v5 exports `auth`, `signIn`, `signOut`, `handlers` — **not** `authOptions`.
- Prisma 7 uses the `prisma-client` generator. Importing from `@prisma/client` still works; the output path is `src/generated/prisma`.
- Tailwind v4 has no `tailwind.config.js` — theme extension lives in CSS.
- `src/generated/prisma` is gitignored — always run `npx prisma generate` after cloning or after a schema change.
- React 19 strict mode double-invokes server actions in development — this is expected.
- shadcn/ui v4+ targets Tailwind v4; do not use the v3 init flags.
